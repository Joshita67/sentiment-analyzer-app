# Sentiment Analysis Full Stack App

A beginner-friendly and scalable Sentiment Analysis application built using React, Spring Boot, and Hugging Face's NLP models.

## Architecture
- **Frontend**: React (TypeScript), Tailwind CSS, Vite.
- **Backend**: Java 17, Spring Boot 3.
- **Database**: MySQL.
- **NLP API**: Hugging Face Inference API (`distilbert-base-uncased-finetuned-sst-2-english`).

---

## 🚀 Setup Instructions

### 1. Database (MySQL) Setup
This app requires a MySQL database called `sentiment_db`.
1. Start your local MySQL server.
2. Run this query to create the database:
   ```sql
   CREATE DATABASE IF NOT EXISTS sentiment_db;
   ```
   *(Note: The Spring Boot app will handle creating tables automatically).*

### 2. Backend Setup (Java / Spring Boot)
1. Open a terminal and navigate to the `backend` folder:
   ```sh
   cd backend
   ```
2. **Configuration**: Open `src/main/resources/application.properties`.
   - Update `spring.datasource.password` if your MySQL root uses a password.
   - **(Optional)** Add a free [Hugging Face Inference API Token](https://huggingface.co/settings/tokens) to `huggingface.api.key`. If you don't add one, the app will gracefully fall back to a mock analysis algorithm so you can test it immediately!
3. **Run the backend**:
   ```sh
   mvn spring-boot:run
   ```
   *The backend will run on `http://localhost:8080`.*

### 3. Frontend Setup (React / Node.js)
1. Open a new terminal and navigate to the `frontend` folder:
   ```sh
   cd frontend
   ```
2. Install dependencies:
   ```sh
   npm install
   ```
3. Start the Vite development server:
   ```sh
   npm run dev
   ```
   *The frontend will run on `http://localhost:5173`.*

## 🎨 Features
- **Real-Time Analysis**: Analyzes text to determine if it is POSITIVE, NEGATIVE, or NEUTRAL.
- **Confidence Scores**: Displays continuous probability scores generated by the transformer model.
- **Educational Insights**: Explainers embedded in the UI to give beginners intuition on how NLP models process data.
- **Fallbacks**: If Hugging Face goes down (or you don't use a key), the app falls back to a smart mock algorithm ensuring demo stability.
